# CoralOS Round

The web oracle (`npm run dev`) is the single-agent product view. This folder is the multi-agent
version: one buyer shops for a TxODDS edge across three seller personas over CoralOS, then settles the
winning delivery through the arbiter-gated Solana escrow on devnet.

```text
buyer-agent
  -> WANT service=txline arg="edge <fixtureId>"

seller-worldcup   -> BID price=0.00045  specialist
seller-fast       -> BID price=0.00065  generalist
seller-premium    -> BID price=0.00085  premium analyst

buyer-agent
  -> AWARD to=seller-worldcup
  <- ESCROW_REQUIRED reference=<bound-order-hash> settlement=arbiter
  -> DEPOSITED settlement=arbiter vault=<vault PDA>
  <- DELIVERED {teams, odds, LLM/fallback read}
  -> ARBITER_RELEASED sig=<devnet tx>
```

The reference is derived from the purchased order fields, and the funded escrow's buyer is the arbiter
vault PDA. The human buyer cannot unilaterally claw funds back after delivery; the configured neutral
arbiter key releases to the seller.

## Run It

Needs Docker for `coral-server` and the repo `.env` with:

- `BUYER_KEYPAIR_B58`, funded with devnet SOL
- `ARBITER_KEYPAIR_B58`, generated by `node scripts/setup.js`
- `WALLET`, the seller payout address
- `TXLINE_API_KEY`, minted with `npm run mint` in `examples/txodds`
- A live LLM key: either `ANTHROPIC_API_KEY`, or `LLM_PROVIDER=openai` plus `OPENAI_API_KEY`

```sh
docker compose up -d coral
cd examples/txodds
npm run coral
```

`round.ts` asks the running proxy for a live fixture id from `/api/board`, starts the buyer plus three
seller personas, and injects `SETTLEMENT_MODE=arbiter` into the agents.

## Watch It

Coral names containers by UUID, so find and tail them:

```sh
docker logs -f $(docker ps -qf ancestor=buyer-agent:0.1.0  | head -1)
docker logs -f $(docker ps -qf ancestor=seller-agent:0.1.0 | head -1)
```

Set `TRACE=1` in `.env` for Coral calls, PDA addresses, and Explorer links.

## Scope, Honestly

- The arbiter is a trusted neutral keypair in this demo. The claim is arbiter-gated settlement between
  buyer and seller, not arbitration without a trusted party.
- If no live LLM provider is funded, the seller returns a deterministic odds read. That keeps the demo
  resilient, but the recorded video should use a live provider.
- The current product is a fair-line read. A true betting edge needs a bookmaker comparison feed.
